TODO Ros: A Sign System for Situated Selection

This is an exploration on UI principles and design. E.g., one such principle is that we should not need to write disjoint boilerplate to get program configuration out of different sources, likei CLI args, JSON input, ENVVARS, or GUI input widgets.

This project has spidered out in ways I really like. Some strands in the web include

  • reflections on intuitionistic mathematics and HCI,
  • a bespoke unit library, and
  • the And_or library (an idiosyncratic derivative of These).

The project is at https://git.sr.ht/~shon/ros
